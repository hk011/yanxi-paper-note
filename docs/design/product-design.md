# 论文解读产品设计文档

## 1\. 产品概述

### 1\.1 产品定位

本产品是一款自动化论文解读工具，旨在帮助科研人员、学生和技术从业者快速理解学术论文内容。通过OCR文档解析模型和多模态大模型解析英文 PDF 论文，结合大语言模型的深度理解能力和联网搜索能力，自动生成结构化、通俗易懂的中文 Markdown 解读笔记。

### 1\.2 核心价值

- **高效解析**：利用 MinerU 模型，精准提取论文中的文本、公式、图表等多模态信息

- **智能解读**：基于 Seed 2\.0 Pro 多模态大模型，生成专业且易懂的论文解读

- **知识增强**：结合联网搜索，补充论文相关的背景知识和最新研究进展

- **结构化输出**：统一的 Markdown 格式的论文笔记输出，包含原图引用（亮点）、要点梳理和深度分析

### 1\.3 目标用户

- 科研人员：快速了解领域内最新研究成果

- 硕博研究生：高效阅读和理解学术文献

- 技术从业者：跟踪前沿技术发展动态

## 3\. MinerU 集成模块

### 3\.1 模块功能

MinerU 是开源的高质量文档解析工具，用于将输入的 PDF 论文转换为结构化的文本、公式和图片数据。本模块负责：

- PDF 文件的上传与预处理

- 调用 MinerU API 进行文档解析

- 解析结果的后处理与格式转换

- 提取文本、公式、图片等多模态信息

### 3\.2 集成方案

1. **API 调用方式**：通过 RESTful API 调用 MinerU 服务

2. **文件传输**：支持本地文件上传和 URL 下载两种方式

3. **异步处理**：对于大文件采用异步解析机制

4. **结果缓存**：对解析结果进行缓存，避免重复解析

### 3\.3 数据处理流程

1. 接收用户上传的 PDF 文件

2. 调用 MinerU API 进行文档解析

3. 解析输出的 JSON 格式结果

4. 提取正文文本、公式 LaTeX 代码、图片文件

5. 将处理后的数据传递给大模型解读模块

### 3\.4 参考文档

- 参考文档：[MinerU API 管理文档](https://mineru.net/apiManage/docs) \- 用于参考 MinerU 的 API 接口调用规范

- 参考文档：[MinerU 输出文件格式说明](https://opendatalab.github.io/MinerU/reference/output_files/#examples) \- 用于参考 MinerU 解析后的输出文件格式和数据结构

## 4\. 联网搜索模块

### 4\.1 模块功能

为了增强论文解读的深度和广度，系统需要通过联网搜索补充以下信息：

- 论文中提到的相关概念和技术背景

- 相关领域的最新研究进展

- 网络上，比如知乎，博客等对于该论文的深度解读

- 如果有，该论文在github上的开源代码

- 业界对该论文的评价和讨论

### 4\.2 搜索策略

1. **背景/概念搜索**：直接搜索相关背景主题和概念词汇

2.**研究进展搜索**：领域关键词中英文搜索，和本篇论文相关的其他前沿论文解读检索，最新的综述解读检索

3.**论文解读检索**：论文标题（英文）或缩写或别称，加上解读和搜索开源检索。比如“transformer解读 知乎 博客”

2. **多轮搜索和思考**：根据解读进度动态发起相关搜索，边思考边搜索

3. **结果筛选**：对搜索结果进行相关性和权威性筛选

4. **信息整合**：将搜索到的信息有机融入解读内容中

### 4\.3 技术实现

采用火山引擎搜索工具作为底层搜索能力，通过 API 方式调用，支持：

- 精准的中文搜索能力

- 学术资源优先检索

- 边搜索边思考

- 搜索结果的结构化返回

- 搜索频次和并发控制

## 5\. 前端设计模块

### 5\.1 设计原则

- **简洁高效**：界面简洁，操作流程清晰

- **流式体验**：采用流式输出，提升用户等待体验

- **助手式交互**：遵循 AI 助手交互设计规范

- **响应式布局**：适配不同屏幕尺寸

### 5\.2 核心功能页面

1. **上传页面**：PDF 文件拖拽/点击上传，上传以后可以预览看到pdf原文

2. **解析页面**：展示解析进度和实时状态，一个进度条，根据api文档，可以看到是解析到哪一页了，前端进行更新。解析完成以后，页面可以在上面进行切换，决定是看原文还是解析后的markdown预览文件。或者进行对比查看（此时左边页面可以往右扩，或者额外弹出之类的，你可以规划一下）。

3. **解读页面**：流式展示解读结果，这里的流式需要查看ant里面流式渲染方案，要好看自然，支持原图查看和图片放大。参考语雀文档里面的图片嵌入那种。

4. **历史页面**：查看历史解读记录，支持重新生成

### 5\.3 关键技术点

1. **流式 Markdown 渲染**：支持大模型输出的实时流式渲染

2. **公式渲染**：支持 LaTeX 公式的完美渲染

3. **图片懒加载**：论文原图的优化加载策略

4. **目录导航**：自动生成解读内容的目录导航

## 6\. Markdown 笔记生成模块

### 6\.1 输出结构

生成的 Markdown 解读笔记包含以下标准化结构：

```Plain
# 论文标题解读
## 一、论文基础信息
- 基本信息（标题（原文和译文）、作者、作者单位（原文和译文）、发表时间和发表的期刊/会议、代码库地址、数据地址等）。基本信息用表格展示。

## 二、背景，动机和结果
- 核心贡献总结
- 研究背景与动机

## 三、核心方法
1.一方面参考论文解读或者你自己对论文的理解进行详情和描述。2.要对论文核心的架构图/方案图/模型图等进行详细的描述。3.必要时候，可以使用图像生成模型，生成有助于用户理解的说明图进行说明和论文的深入解读。图像生成模型可以包装成大模型可以使用的MCP工具,和网页搜索一样，边思考边调用。
- 技术架构详解
- 关键算法说明
- 创新点分析

## 四、实验结果
- 实验设置
- 主要结果分析。可以直接使用论文里图表展示并解析。
- 对比实验解读。可以画表。

## 四、总结与展望
- 研究价值评估
- 局限性分析
- 未来研究方向

## 五、扩展阅读
- 相关论文推荐
- 参考资料链接
```

### 6\.2 原图引用策略

- 所有论文中的重要图表在解读中对应位置引用

- 图片保留原始分辨率，支持点击放大查看

- 图表标题和说明文字完整保留

- 公式采用 LaTeX 格式渲染

## 7\. 助手式交互优化模块

笔记生成以后，左侧有一个助手式的交互问答（可以有一些示例问题的推荐），可以让用户在这里和ai进行关于论文内容的问答，上下文就是论文笔记内容以及可能用到的论文原始内容之类的。并且这里用户的问题可能很宽泛，所以需要有联网搜索的能力。可以效仿deepseek的官方网站，手点来开启或关闭深度思考和联网搜索等功能。然后可以上传图片等。
此外，无论是笔记生成还是这个问答交互，都可以自己切模型，前端下滑展开切换即可，可以效仿豆包的交互和简洁的界面。
（多轮问答的时候，最好是实时统计上下文，跟claude code或者cursor类似，有个上下文的⭕️，展示已经用了多少，还有多少），多轮问答的上下文传输，上一轮的思考过程是不用传输的，不过工具调用的结果是要传输的，你可以自己搜索一下怎么处理多轮问答！用户多轮问答以后，可以支持笔记优化，就是用户有不明白的，或者需要延伸的才会问答，可以把问答涉及到的知识融合到现有的笔记中，一般是细化，扩展，对比，总结等等形式和现有笔记进行融合，你也可以多思考这个时候prompt怎么写。
